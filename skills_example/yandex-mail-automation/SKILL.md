# Яндекс.Почта — оптимизированный сценарий

## Предпочтительный путь (один вызов, меньше токенов)

Если у пользователя есть **логин, пароль и формулировка запроса** (тема/ключевые слова):
- Вызови **yandex_mail_fetch**(login, password, subject_query). Тул сам: открывает mail.yandex.ru, вводит логин/пароль, при 2FA вернёт `status=needs_2fa` — тогда запроси код у пользователя и вызови снова с `two_factor_code`. Результат — текст писем в `message_snapshot`.
- Не дублируй эти шаги через browser_navigate/browser_type вручную — это лишние токены и шаги.

## Когда использовать только browser_* (fallback)

Используй сырые browser-тулы только если:
- тул yandex_mail_fetch недоступен (не включён или ошибка), или
- нужен нестандартный сценарий (не поиск по теме, а конкретная папка/метка и т.п.).

Канонический порядок (интерфейс может меняться — всегда сначала снапшот):
1. browser_navigate("https://mail.yandex.ru/")
2. browser_snapshot() — по refs найди кнопку «Войти», поля логин/пароль, поиск, список писем.
3. Клик «Войти» → ввод логина → Enter → ввод пароля → Enter.
4. Если появился экран 2FA — запроси код у пользователя, введи в поле кода, Enter.
5. Для поиска: найди поле поиска по snapshot, введи запрос, Enter.
6. Чтобы прочитать письмо: по snapshot кликни по нужной строке списка, затем browser_snapshot(full=True) для текста письма.

Сессия браузера общая в рамках task_id — повторный логин не нужен, пока не закрыли браузер.

## Важно

- **Капча/блок:** при детекте (captcha, «подтвердите») — либо включи Browserbase (облако с авто-капчей), либо верни пользователю снапшот/скрин и попроси пройти вручную.
- **Селекторы меняются:** не завязывайся на конкретные ref (@eN). Всегда делай snapshot перед кликом/вводом и выбирай элемент по тексту/роли из текущего снапшота.
